Power Tools 파워 툴즈

Six focused features that extend the pi agent's capabilities — URL fetching, editor state management, artifact persistence, structured commit metadata, branded startup, and real-time goal/todo tracking. pi 에이전트의 기능을 확장하는 6개의 집중 기능 — URL 가져오기, 에디터 상태 관리, 아티팩트 지속성, 구조화된 커밋 메타데이터, 브랜디드 시작 화면, 실시간 목표/할 일 추적.

● WebFetch

URL → Markdown conversion tool for agents. Fetches any URL, converts HTML to clean Markdown using Mozilla Readability + Turndown, and caches results for fast repeated access. 에이전트를 위한 URL → Markdown 변환 도구. 모든 URL을 가져와 Mozilla Readability + Turndown으로 HTML을 깔끔한 Markdown으로 변환하고, 빠른 반복 접근을 위해 결과를 캐시합니다.

Parameter파라미터 Type타입 Required필수 Default기본값

Cache 캐시

In-memory LRU cache. 100 entries max, 15-minute TTL. Cache key is derived from JSON.stringify({url, mode, scripts}) where mode is "full" (default) or "full" when raw is true. 인메모리 LRU 캐시. 최대 100개 항목, 15분 TTL. 캐시 키는 JSON.stringify({url, mode, scripts})에서 파생됩니다. 여기서 mode는 기본값 "auto" 또는 raw가 true일 때 "full"입니다.

Limits 제한

  • MAX_CONTENT_SIZE = 10 MB — checked via content-length header and actual byte count content-length 헤더와 실제 바이트 수로 확인
  • FETCH_TIMEOUT_MS = 30 s — hard abort via AbortController AbortController를 통한 강제 중단

User Agent 사용자 에이전트

Mozilla/5.0 (compatible; WebFetchTool/1.0)

Extraction Modes 추출 모드

Mode모드 Trigger트리거 Behavior동작

Truncation 자르기

When maxLength is set and content exceeds it, the returned copy is sliced and suffixed with: maxLength가 설정되고 콘텐츠가 초과하면 반환된 사본은 잘리고 다음 접미사가 붙습니다:

... (truncated)

The original content stored in cache is not truncated. 캐시에 저장된 원본 콘텐츠는 자르지 않습니다.

■ Editor Stash

Session-scoped single-slot stash for editor text. Save a snippet, clear the editor, restore it later — all without leaving the editor. 에디터 텍스트를 위한 세션 범위의 단일 슬롯 스태시. 스니펫을 저장하고, 에디터를 지우고, 나중에 복원 — 에디터를 떠나지 않고 모두 가능합니다.

★ Artifacts

Structured per-run artifact directories for subagent outputs. Each subagent invocation gets an isolated directory tree for its output, progress, and declared read files. 서브에이전트 출력을 위한 구조화된 실행별 아티팩트 디렉토리. 각 서브에이전트 호출은 출력, 진행, 선언된 읽기 파일을 위한 격리된 디렉토리 트리를 가져옵니다.

Path Guards 경로 가드

All user-supplied paths are validated before resolution: 모든 사용자 제공 경로는 해결 전에 검증됩니다:

  • Rejects absolute paths (/etc/passwd) 절대 경로 거부 (/etc/passwd)
  • Rejects home shortcuts (~, ~/...) 홈 단축키 거부 (~, ~/...)
  • Rejects paths that escape the run directory (../../etc) 런 디렉토리를 벗어나는 경로 거부 (../../etc)
  • Rejects paths that escape the workspace (for declared reads) 워크스페이스를 벗어나는 경로 거부 (선언된 읽기용)

Segment Sanitization 세그먼트 정화

sanitizeSegment() replaces non-alphanumeric characters (except ., _, -) with -, then strips leading and trailing dashes. Falls back to "run" if the result is empty. sanitizeSegment()은 영숫자가 아닌 문자(., _, - 제외)를 -로 교체하고, 선행 및 후행 대시를 제거합니다. 결과가 비어있으면 "run"을 사용합니다.

▶ Welcome Banner

Šváb Pi branded header that replaces the generic pi startup listing. Shows the ASCII art logo and quick-start hints when the session begins. 일반적인 pi 시작 목록을 대체하는 Šváb Pi 브랜디드 헤더. 세션 시작 시 ASCII 아트 로고와 빠른 시작 힌트를 표시합니다.

/welcome Command /welcome 명령

Input입력 Effect효과

on, show, restore Show the banner배너 표시
(bare) /welcome Toggle visibility가시성 토글

Interaction with /setup /setup과의 상호작용

/setup sets quietStartup: true so the pi agent's generic startup listing is suppressed, making the Šváb Pi banner the main startup surface. /setup이 quietStartup: true를 설정하여 pi 에이전트의 일반적인 시작 목록이 억제되고, Šváb Pi 배너가 주 시작 화면이 됩니다.

Interaction with /init /init과의 상호작용

/init also configures banner visibility as part of the initial project setup flow. /init도 초기 프로젝트 설정 흐름의 일부로 배너 가시성을 구성합니다.

◐ Goal & Todo Progress

Real-time goal, subgoal, evidence, and todo state displayed in the TUI footer. The footer reflects the durable /goal runtime and structured todo ledger rather than parsing legacy files. TUI 푸터에 표시되는 실시간 목표, 하위 목표, 증거, 할 일 상태입니다. 푸터는 레거시 파일을 파싱하지 않고 지속형 /goal 런타임과 구조화된 할 일 원장을 반영합니다.

Goal Runtime State 목표 런타임 상태

/goal tracks active goals, active subgoals, evidence, blockers, and verifier receipts. The footer renders this structured state directly so progress survives compaction and session replay. /goal은 활성 목표, 활성 하위 목표, 증거, 차단 요소, 검증 영수증을 추적합니다. 푸터는 이 구조화된 상태를 직접 렌더링하므로 컴팩션과 세션 리플레이 이후에도 진행 상황이 유지됩니다.

/reset-phase Command /reset-phase 명령

Resets the transient workflow phase when the UI needs a clean active-state display. UI가 깨끗한 활성 상태 표시를 필요로 할 때 임시 워크플로 단계를 재설정합니다.

Footer Integration 푸터 통합

The footer reads structured goal and todo state, then refreshes when tool calls or session replay update that state. 푸터는 구조화된 목표 및 할 일 상태를 읽고, 도구 호출이나 세션 리플레이가 상태를 갱신하면 다시 렌더링합니다.
