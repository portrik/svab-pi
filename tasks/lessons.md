# Lessons

- When implementing terminal shortcuts, do not compare only raw control bytes. Use Pi TUI `matchesKey`/keybinding helpers and test encoded terminal variants (CSI-u / modifyOtherKeys) because visible shortcut hints can render while real key events arrive in a different format.
- Before making or committing changes, confirm the target repository path when multiple repos are in context. Do not continue working in a repo just because it appeared in compacted history; explicitly verify it matches the user's current "our repo" reference.
- For UI state that survives reload, always test both live event handling and `session_start` history reconstruction. A fix that only handles new tool events can still fail after reload if stale custom snapshots are restored.
- For milestone tracking, test every real lifecycle signal separately: artifact/state discovery, active planning start, execution, validation, completion, and reload. Do not assume detecting the milestone list implies active status transitions work.
- For visual UI fixes, do not invent colors from semantic foreground values unless the user explicitly approves the palette. Preserve the original/intended palette or ask for a screenshot/reference before changing color semantics.
- Before patching UI state tracking based on a live screenshot, verify whether the visible state is actually expected for the current workflow phase. Do not treat "not visibly active yet" as a bug until the lifecycle source of truth (state.md/session event/current phase) confirms it.
- **Verify API existence before planning dependent features.** `ctx.getThinkingLevel()` was assumed available based on `ThinkingLevelSelectEvent` in the SDK, but the runtime `ExtensionContext` did not expose it. Always grep the installed `.d.ts` for the exact method name before including it in a plan.
- **Expanding an interface requires updating every test stub.** Adding `getGitStats`, `getThinkingLevel`, and `getModelInfo` to `FooterContext` broke 4 test files that constructed `SvabFooter` with partial stubs. When extending shared types, immediately grep for all constructor call sites and update stubs in the same commit.
- **Gracefully degrade for missing APIs instead of crashing.** The footer initially threw `TypeError: ctx.getThinkingLevel is not a function` because it was called unconditionally. Always wrap optional context accessors with fallback values (`() => undefined`) so the UI renders even when upstream APIs are absent or outdated.
- **Width-sensitive tests need wider buffers when adding segments.** The compact preset test failed at width 100 because the new `logo + model + thinking + path + git` line exceeded the limit. When adding segments to powerline layouts, bump test widths proportionally or verify `truncateToWidth` behavior explicitly.
- **`/goal` without an explicit target means the whole active goal, not just the current subgoal.** Auto prompts must say to continue across subgoal PASS receipts, then request goal-level completion, and stop only after the active goal itself receives reviewer-verifier PASS.
